# 分页功能需求文档（基于 UI 截图梳理）

> 本文基于用户提供的两张截图，梳理分页模式的功能点与逻辑。暂不涉及代码实现。

---

## 一、总体架构

### 1.1 两种页面模式

| 模式 | 预览形态 | 导出形态 |
|------|---------|---------|
| **整页（full）** | 一张无限延伸的长卡片，高度随内容自动增长 | 导出为一张长图（现有逻辑） |
| **分页（paged）** | 按固定比例切成多张卡片页，纵向排列 | 可逐页下载，或打包导出 |

### 1.2 模式切换

- 切换入口：左侧面板「自定义」→「页面模式」→「整页 / 分页」两个按钮
- 切换到「分页」时，下方展开「卡片比例」选择器
- 切换到「整页」时，隐藏「卡片比例」选择器

## 二、卡片比例（Aspect Ratio）

### 2.1 可选比例

从截图中识别出 8 种比例：

| 比例 | 宽高关系 | 适用场景 |
|------|---------|---------|
| 1:1 | 正方形 | 头像、封面 |
| 3:4 | 竖屏偏短（截图中的默认选中项）| 小红书、朋友圈 |
| 3:5 | 竖屏中等 | 笔记、短文 |
| 2:3 | 竖屏中等 | 通用卡片 |
| 1:2 | 竖屏较长 | 长内容 |
| 9:16 | 手机屏幕比例 | 短视频封面、全屏海报 |
| 9:19 | 超长屏 | 手机壁纸 |
| 9:21 | 极长屏 | 特殊场景 |

### 2.2 比例的作用

- 确定**每一页卡片的高度**：`高度 = 宽度 × 比例`
- 当前卡片固定宽度为 440px（从现有代码中），因此各比例对应的单页高度：
  - 1:1 → 440px
  - 3:4 → 587px
  - 3:5 → 733px
  - 2:3 → 660px
  - 1:2 → 880px
  - 9:16 → 782px
  - 9:19 → 929px
  - 9:21 → 1027px

### 2.3 默认比例

- 默认选中 **3:4**（截图中的绿色高亮项）

## 三、分页预览区（核心交互）

### 3.1 页面容器

- 预览区纵向排列多张卡片页
- 每页之间留有间距（截图中约为 16-24px 的灰色间隙）
- 每页左下角显示**页码**（如 "2"）

### 3.2 单页结构

每一页都包含三个固定区域 + 一个动态区域：

```
┌─────────────────────────────┐
│  01          妙记文卡        │  ← 页头（Header）
│  2026/06                     │
├─────────────────────────────┤
│                             │
│      正文内容（动态填充）      │  ← 正文（Body）
│      按高度自动分页           │
│                             │
├─────────────────────────────┤
│  妙记文卡    极简易用的图文   │  ← 页脚（Footer）
│  妙记文卡，高颜值卡片制作神器  │
└─────────────────────────────┘
   2                           ← 页码（左下角）
   📋  ⬇️                       ← 操作按钮（右上角）
```

### 3.3 页头（Header）每页重复

从截图中识别出的页头结构：

- **左侧**：大号日期数字（如 "01"），下方小字显示年/月（如 "2026/06"）
- **右侧**：固定文字（如 "妙记文卡"）
- 使用现有的 `header` 配置项（styleId、date、text、text2、avatarUrl），但渲染方式需要适配分页样式

> 注意：截图中的分页页头和现有整页模式的页头样式不同。分页模式的页头是「大日期 + 年月 + 右上角文字」的布局，类似现有 style2 的变体。

### 3.4 页脚（Footer）每页重复

- 使用现有的 `footer` 配置项
- 截图中显示：左文 + 右文 + 底部小字（二维码副标题）

### 3.5 页码

- 位置：每页左下角
- 格式：纯数字（1, 2, 3...）
- 字号：较小，灰色

## 四、单页操作按钮

每页右上角提供两个操作按钮（截图中可见）：

| 按钮 | 图标 | 功能 |
|------|------|------|
| **复制** | 📋（复制/文档图标）| 将该页内容复制到剪贴板（或复制该页生成的图片） |
| **下载** | ⬇️（下载图标）| 单独导出该页为 PNG/JPG 图片 |

### 4.1 复制功能

- 点击「复制」→ 将该页渲染为图片 → 写入剪贴板
- 需要 toast 提示「已复制到剪贴板」

### 4.2 单页下载功能

- 点击「下载」→ 仅导出当前这一页为图片
- 文件名格式：`墨迹文卡_第2页_20260602.png`
- 清晰度继承全局导出设置（高清/超清/极致/专业）

## 五、内容分页算法（核心难点）

### 5.1 分页目标

将无限长度的 HTML 内容，按「单页固定高度」切成若干段，每段放入一页。

### 5.2 分页原则

1. **不在文字中间截断**：分页点应选择在段落之间、列表项之间、标题之后
2. **图片不跨页**：如果一张图片在当前页放不下，整张图片移至下一页
3. **代码块不跨页**：如果代码块在当前页放不下，整块移至下一页
4. **表格不跨页**：如果表格在当前页放不下，整块移至下一页
5. **引用块不跨页**：同上
6. **最小内容保护**：每页至少应有一行文字（避免空白页）

### 5.3 分页策略建议

**策略 A：DOM 逐块测量（推荐）**

1. 将内容渲染到一个隐藏的「测量容器」中
2. 按块（段落、列表、图片、代码块等）逐个测量高度
3. 累加高度，超过单页上限时，当前块放入下一页
4. 重复直到所有内容分配完毕

**策略 B：CSS 打印分页模拟**

1. 使用 `page-break-after: avoid` / `page-break-inside: avoid` 等 CSS 属性
2. 利用浏览器的打印引擎自然分页
3. 但控制精度较低，不太适合精确卡片比例

### 5.4 边界情况

| 场景 | 处理方案 |
|------|---------|
| 某一页内容很少（只剩一行字）| 允许，不强制合并到上一页 |
| 某一页完全空白 | 不应该出现；如果真出现，删除该页 |
| 单张图片就超过一页高度 | 图片按比例缩放至单页内（保持最大宽度） |
| 内容总长度不足一页 | 只显示一页，页高仍为固定比例（下方留白） |
| 页头 + 页脚 + 一行文字就超出一页 | 缩小字体或允许轻微溢出（极端情况） |

## 六、导出逻辑

### 6.1 整页模式导出（现有逻辑，保持不变）

- 一张长图
- 可选 PNG / JPG
- 可选清晰度 ×3 / ×4 / ×5 / ×6

### 6.2 分页模式导出（新增）

**方式一：逐页导出（从单页按钮触发）**
- 点击某页的「下载」按钮，仅导出该页
- 使用与全局相同的格式和清晰度设置

**方式二：批量导出（从顶部「导出」按钮触发）**
- 需要新增「导出全部页」选项
- 可能的交互：
  - 导出为 ZIP（内含多张 PNG/JPG）
  - 或逐张自动下载（浏览器限制：不能同时下载太多张）

### 6.3 导出实现注意

- 分页导出时，每一页都是一个独立的 DOM 片段
- 需要单独调用 `html-to-image` 对每一页生成图片
- 预热策略（scale=1 预热）对每一页都要做，但可以共享缓存

## 七、状态管理变更

### 7.1 新增/变更字段

```
CardConfig {
  pageMode: 'full' | 'paged'   // 已有字段，需接入预览和导出
  + aspectRatio: string         // 新增：卡片比例，如 '3:4'
  ...其余字段保持不变
}
```

### 7.2 条件渲染规则

| 条件 | 整页模式 | 分页模式 |
|------|---------|---------|
| 显示「卡片比例」选择器 | ❌ 隐藏 | ✅ 显示 |
| 预览区显示页码 | ❌ 无 | ✅ 每页左下角 |
| 预览区显示单页操作按钮 | ❌ 无 | ✅ 每页右上角 |
| 页头页脚 | 仅在首尾各一个 | 每页重复 |
| 导出方式 | 一张长图 | 逐页或批量 |

### 8.1 卡片比例选择器

- 选中态：绿色边框 + 绿色文字
- 默认选中：3:4

- 默认状态：半透明/隐藏
- Hover 状态：显示完整按钮
- 位置：每页右上角
